# HTML Report Scope

This document defines the first implementation segment for HTML reports. It is intentionally narrow: add a static report artifact that extends the existing local report pipeline without introducing a dashboard, server, JavaScript framework, or external assets.

## Goal

`agentprof report generate` should produce a local HTML report alongside the existing Markdown, JSON, and optional SVG artifacts.

Expected outputs:

```text
.agentprof/reports/
  <report-id>.md
  <report-id>.json
  <report-id>.html
  <report-id>-multi-agent-waste.svg  # only when multi-agent waste issues exist
```

## Non-Goals

- No hosted web app.
- No local web server.
- No JavaScript framework.
- No external CDN, fonts, scripts, or image dependencies.
- No new analyzer behavior.
- No changes to privacy defaults.
- No direct DuckDB browsing UI.

## Source Data

The HTML renderer should use the same report payload currently written to JSON by `agentprof/report/runner.py`:

- `summary`
- `issues`
- issue `evidence`
- `cost_ledger`
- optional `summary.artifacts.multi_agent_waste_svg`

The HTML output should not read directly from DuckDB. Keeping HTML generation downstream of the existing report payload avoids a second reporting data model.

## UX Shape

The first HTML version should be a polished static report, not a dashboard.

Required sections:

- Header with project, report ID, and generated timestamp.
- Summary cards for issue count, evidence count, affected traces/spans, wasted cost, and potential savings.
- Visuals section when an optional SVG artifact exists.
- Issue cards with severity, confidence, recommendation, recommended tests, and evidence.
- Cost ledger table.
- Empty-state messaging when no issues or costs exist.

## Safety Requirements

- Escape all user-controlled or trace-controlled strings with HTML escaping.
- Keep generated HTML inside `.agentprof/reports/`.
- Reuse the existing report ID validation.
- Reuse the existing safe artifact-show path handling when CLI support is added.
- Keep the file self-contained except for sibling SVG artifacts generated by AgentProf.

## Acceptance Criteria

- `agentprof report generate --report-id latest` writes `.md`, `.json`, and `.html` files.
- The HTML report renders all existing Markdown report information.
- Multi-agent SVG visuals are linked or embedded when present.
- `agentprof report show latest --format html` prints the HTML artifact after CLI support lands.
- Tests cover escaping, empty reports, issue details, cost rows, optional visuals, and CLI artifact safety.
